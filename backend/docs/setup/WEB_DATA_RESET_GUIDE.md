# 🌐 网页版数据重置和重新爬取指南

## 📋 功能概述

网页版数据重置功能提供了一个直观的Web界面，让您可以安全地删除现有章节数据并从网站重新爬取最新内容。

## 🚀 快速开始

### 第一步：访问数据重置页面

打开浏览器，访问以下地址：
```
http://localhost:8000/novels/data/reset/
```

或者通过小说详情页面导航：
1. 访问小说详情页：`http://localhost:8000/novels/1/`
2. 点击 **"数据重置"** 按钮

### 第二步：查看当前状态

页面会显示当前数据库的统计信息：
- 📚 **小说数量**: 当前数据库中的小说总数
- 📖 **章节数量**: 当前数据库中的章节总数
- 📦 **备份状态**: 是否有可用的备份文件
- ⏱️ **操作状态**: 当前操作状态

## 🔧 操作步骤

### 方式1：逐步操作（推荐）

#### 1. 创建备份
```bash
点击 "创建备份" 按钮
```
- 系统会自动创建当前所有数据的备份
- 备份文件保存在 `backend/backup/` 目录
- 文件名格式：`novels_backup_YYYYMMDD_HHMMSS.json`

#### 2. 删除现有数据
```bash
点击 "删除数据" 按钮
```
- 系统会安全地删除所有章节数据
- 使用事务保护，确保操作的原子性
- 保留小说基本信息

#### 3. 重新爬取数据
```bash
点击 "重新爬取" 按钮
```
- 系统会从网站重新下载所有章节
- 使用激进模式，爬取速度更快
- 实时显示进度信息

### 方式2：一键完整重置

```bash
点击 "完整重置" 按钮
```

系统会自动执行：
1. 📦 **创建备份** - 备份现有数据
2. 🗑️ **删除数据** - 删除所有章节
3. 🕷️ **重新爬取** - 从网站下载最新数据

## ⚙️ 高级设置

### 爬取速度控制

网页版使用激进配置，默认设置：
```json
{
  "request_limits": {
    "daily_limit": 200,
    "hourly_limit": 100
  },
  "delays": {
    "download_delay": 3,
    "page_load_delay": 5,
    "request_interval": 1
  }
}
```

### 自定义配置

如需调整速度，编辑配置文件：
```bash
# 编辑配置
nano backend/safe_crawler_config.json

# 修改参数值
"download_delay": 2,      // 更快的间隔
"page_load_delay": 3,     // 更快的等待
"request_interval": 0.5   // 更短的间隔
```

## 🔍 故障排除

### 问题：页面无法访问
```
解决步骤：
1. 确保Django服务器正在运行
2. 检查URL路径是否正确
3. 确认用户权限设置
```

### 问题：操作失败
```
解决步骤：
1. 查看操作日志中的错误信息
2. 检查网络连接
3. 确认网站是否可访问
4. 尝试刷新页面重新操作
```

### 问题：爬取速度慢
```
解决步骤：
1. 检查当前配置参数
2. 调整 download_delay 参数
3. 减少 page_load_delay 值
4. 考虑修改 safe_crawler_config.json
```
