# Django Admin 爬虫功能使用指南

## 🚀 快速开始

### 1. 访问管理界面

1. 确保Django服务器正在运行：
   ```bash
   python manage.py runserver 8000
   ```

2. 在浏览器中打开：`http://127.0.0.1:8000/admin/`

3. 使用管理员账号登录

### 2. 爬取小说章节

#### 方法一：从小说来源关系页面爬取

1. **进入小说来源关系管理页面**
   - 在Django Admin首页，点击 `Novel source relations`
   - 或直接访问：`http://127.0.0.1:8000/admin/novels/novelsourcerelation/`

2. **选择要爬取的小说**
   - 在列表中找到您想要爬取的小说
   - 勾选左侧的复选框（可以选择多个）

3. **执行爬取操作**
   - 在页面底部的"Action"下拉菜单中选择：`🕷️ Pachong爬取小说`
   - 点击 `Go` 按钮

4. **查看爬取结果**
   - 系统会显示爬取进度和结果
   - 成功后会显示新增章节数量

#### 方法二：从小说来源页面爬取

1. **进入小说来源管理页面**
   - 在Django Admin首页，点击 `Novel sources`
   - 或直接访问：`http://127.0.0.1:8000/admin/novels/novelsource/`

2. **选择爬虫类型**
   - 选择来源后，在Action中可以选择：
     - `🚀 Pachong快速爬虫` - 快速获取书籍列表
     - `📚 Pachong详细爬虫` - 详细爬取章节内容

## 📝 操作示例

### 示例1：爬取和图书网小说

2. 找到和图书网的小说记录（URL包含hetushu.com）

## ⚠️ 注意事项

### 网络连接
- 确保网络连接稳定
- 某些网站可能有访问限制
- 建议在网络较好的时候进行爬取

### 爬取频率
- 避免过于频繁的爬取请求
- 尊重网站的robots.txt规则
- 建议适当间隔进行爬取

### 数据备份
- 重要数据请及时备份
- 爬取前可以导出现有数据
- 定期检查数据完整性

## 🔧 故障排除

### 常见问题

**Q: 爬取失败怎么办？**
A: 
1. 检查网络连接
2. 确认目标网站是否可访问
3. 查看Django日志获取详细错误信息
4. 尝试重新爬取

**Q: 章节重复怎么办？**
A: 系统有自动去重功能，重复章节不会被保存

**Q: 爬取速度慢怎么办？**
A: 
1. 检查网络速度
2. 减少并发爬取的小说数量
3. 选择网络较好的时间段

**Q: 某些章节爬取不到？**
A: 
1. 可能是网站结构变化
2. 可能有反爬虫机制
3. 可以尝试手动添加章节
