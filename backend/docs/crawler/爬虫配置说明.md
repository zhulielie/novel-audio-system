# 🕷️ Django Admin 爬虫配置详解

## 📋 当前爬虫行为说明

### 🔍 自动检测机制

当您在Django Admin中使用 `🕷️ Pachong爬取小说` 功能时，系统会：

1. **网站识别**：
   - 如果URL包含 `hetushu`，使用**和图书网专用爬虫**
   - 其他网站使用**通用爬虫**

2. **智能去重**：
   - 自动检查章节标题是否已存在
   - **不会覆盖**已有章节
   - 只下载新章节

3. **章节排序**：
   - 自动获取当前最大排序号
   - 新章节按顺序递增排序

## ⚙️ 当前配置参数

### 📊 章节数量控制

**当前行为**：
- ✅ **无限制**：爬取所有可用章节
- ✅ **智能去重**：跳过已存在的章节
- ✅ **增量更新**：只添加新章节

### 🔄 覆盖设置

**当前行为**：
- ❌ **不覆盖**：已存在的章节不会被重新下载
- ✅ **保护数据**：避免意外覆盖用户编辑的内容
- ✅ **安全模式**：确保数据完整性

### 📍 起始位置

**当前行为**：
- 🚀 **从头开始**：总是从第一章开始检查
- 📈 **顺序添加**：新章节添加到现有章节之后
- 🔢 **自动编号**：系统自动分配章节号和排序号

## 🛠️ 高级配置选项

### 🌐 网站特定配置

#### 和图书网 (hetushu.com)
```json
{
  "request_config": {
    "timeout": 30000,
    "delay_between_requests": 3000,
    "retry_count": 3,
    "retry_delay": 5000
  },
  "content_processing": {
    "title_cleanup": {
      "remove_patterns": ["\\s*第\\d+章\\s*"],
      "replace_patterns": {}
    }
  }
}
```

#### 通用网站配置
```json
{
  "global_settings": {
    "log_level": "info",
    "max_concurrent_downloads": 5,
    "default_timeout": 25000
  }
}
```

## 🎯 如何自定义爬取行为

### 方法1：修改配置文件

编辑 `downloader_config.json`：

```json
{
  "sites": {
    "hetushu": {
      "request_config": {
        "delay_between_requests": 2000,  // 减少延迟
        "timeout": 20000,               // 减少超时
        "retry_count": 2                // 减少重试次数
      }
    }
  }
}
```

### 方法2：Django Admin参数

**当前支持的操作**：

1. **选择性爬取**：
   - 在列表中选择特定的小说
   - 批量操作多个小说

2. **分批处理**：
   - 选择少量小说进行测试
   - 确认效果后再批量处理

## 📈 爬取统计信息

### 📊 自动更新的统计

- **sync_count**：爬取次数
- **chapter_count**：章节数量
- **last_sync_at**：最后爬取时间
- **crawl_count**：来源爬取次数
- **last_crawl_at**：来源最后爬取时间

## ⚠️ 注意事项

### 🚫 当前限制

1. **无章节数量限制**：
   - 会爬取所有可用章节
   - 可能耗时较长
   - 建议分批处理

2. **无起始章节设置**：
   - 总是从第一章开始检查
   - 依赖智能去重避免重复

3. **无覆盖选项**：
   - 不能强制更新已有章节
   - 保护用户编辑的内容

### 💡 最佳实践

1. **测试先行**：
   - 先选择1-2个小说测试
   - 确认爬取效果
   - 再进行批量操作

2. **网络稳定**：
   - 确保网络连接稳定
   - 避免在网络高峰期爬取

3. **分批处理**：
   - 每次选择5-10个小说
   - 避免一次性处理过多

4. **监控日志**：
   - 关注Django Admin的消息提示
   - 查看服务器日志了解详情
