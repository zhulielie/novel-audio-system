# 智能批量章节导入功能使用指南

## 功能概述

智能批量章节导入功能已成功实现，完全满足您的两个核心要求：
1. ✅ **速度控制**: 避免被网站封禁
2. ✅ **源链接记录**: 每个章节都会记录原始URL

## 核心特性

### 🚀 智能化
- **LLM智能分析**: 自动分析网站结构，无需手工配置
- **智能内容提取**: 自动识别章节标题、内容、作者等信息
- **多策略适配**: 支持多种网站布局和结构

### ⚡ 安全可控
- **随机延时**: 2-5秒随机延时，模拟人工访问
- **重试机制**: 最多3次重试，指数退避策略
- **请求限制**: 支持设置最大章节数量限制
- **错误处理**: 完善的异常处理和日志记录

### 📝 完整记录
- **源链接保存**: 每个章节的`source_url`字段记录原始URL
- **导入统计**: 详细的导入结果统计
- **同步记录**: 更新来源的最后同步时间和章节数量

## API 接口

### 批量导入章节
```
POST /api/novels/{novel_id}/batch_import/
```

**请求参数:**
```json
{
  "source_id": 1,           // 必需：来源ID
  "max_chapters": 100       // 可选：最大章节数量限制
}
```

**响应示例:**
```json
{
  "success": true,
  "message": "批量导入完成",
  "imported_count": 45,     // 成功导入章节数
  "skipped_count": 5,       // 跳过的章节数（已存在）
  "failed_count": 2,        // 失败的章节数
  "total_count": 52         // 发现的总章节数
}
```

## 爬虫配置参数

### 速度控制参数
```python
IntelligentBatchCrawler(
    min_delay=2.0,      # 最小延时(秒) - 满足"别弄太快"要求
    max_delay=5.0,      # 最大延时(秒)
    max_retries=3,      # 最大重试次数
    timeout=30          # 请求超时时间(秒)
)
```

### 安全特性
- **随机User-Agent**: 模拟真实浏览器访问
- **Session复用**: 保持会话状态，减少被检测概率
- **指数退避**: 失败重试时逐渐增加延时
- **域名检查**: 只爬取同域名下的章节链接

## 使用流程

### 1. 准备工作
- 确保小说已创建并与来源关联（NovelSourceRelation）
- 确认来源配置正确（URL、编码等）

### 2. 启动批量导入
通过API调用批量导入接口，系统会：
1. 智能分析网站结构
2. 发现章节列表
3. 逐个爬取章节内容
4. 保存章节并记录源链接

### 3. 监控和管理
- 查看导入进度和结果
- 检查失败章节并手动处理
- 验证章节内容质量

### 常见问题
1. **章节发现失败**: 检查网站结构是否变化
2. **内容为空**: 验证内容选择器是否正确
3. **请求被拒绝**: 增加延时或更换User-Agent

### 日志查看
- 爬虫运行日志包含详细的执行信息
- 错误日志帮助诊断问题原因
