# 📚 小说管理页面爬取功能指南

## 🎯 功能概述

Django Admin的小说管理页面现在新增了**爬取功能**，让你可以：
- ✅ 在列表页面直接查看爬取状态
- 🔗 设置和编辑来源链接
- 🕷️ 直接从来源链接爬取小说
- 🔍 自动检测来源链接
- 📊 实时监控爬取进度

## 🚀 快速开始

### 第一步：访问小说管理页面

打开浏览器，访问：
```
http://localhost:8000/admin/novels/novel/
```

### 第二步：查看新增的列

在小说列表页面，你会看到新增的列：

| 列名 | 说明 |
|------|------|
| **来源链接** | 显示设置的来源URL（可点击访问） |
| **爬取状态** | 显示当前小说的爬取状态 |
| **章节数量** | 显示已爬取的章节数 |

### 第三步：设置来源链接

#### 方式1：手动设置
1. 点击小说标题进入编辑页面
2. 在"爬取配置"部分找到"来源链接"字段
3. 输入第一章或目录页面的URL
4. 保存更改

#### 方式2：自动检测
1. 在列表页面选中要检测的小说
2. 选择"🔍 自动检测来源链接"操作
3. 系统会自动搜索并设置合适的来源链接

## 📋 爬取状态说明

| 状态 | 颜色 | 说明 |
|------|------|------|
| **未设置** | 灰色 | 尚未设置来源链接 |
| **待爬取** | 黄色 | 已设置链接但未开始爬取 |
| **部分** | 蓝色 | 已爬取少量章节（<10章） |
| **已爬取** | 绿色 | 已爬取大量章节（≥10章） |

## 🕷️ 爬取操作

### 单个小说爬取

#### 方式1：从编辑页面爬取
1. 进入小说编辑页面
2. 确保"来源链接"已设置
3. 使用页面上的"🕷️ 从来源链接爬取"按钮

#### 方式2：从列表页面批量爬取
1. 在列表页面选中要爬取的小说
2. 选择"🕷️ 从来源链接爬取"操作
3. 点击"执行"按钮

### 批量操作
1. 选中多个小说
2. 选择相应的操作：
   - **🕷️ 从来源链接爬取**：爬取已设置链接的小说
   - **🔍 自动检测来源链接**：为小说自动查找来源链接

## 🔍 自动检测功能

系统支持自动检测以下网站的来源链接：

### 支持的网站
- **和图书** (hetushu.com) - 优先推荐
- **起点中文网** (qidian.com) - 实验性支持
- **纵横中文网** (zongheng.com) - 实验性支持

### 检测逻辑
1. 使用小说标题进行搜索
2. 验证搜索结果是否匹配
3. 自动设置最合适的链接

### 使用示例
```python
# 系统会自动搜索：
# https://www.hetushu.com/search.php?searchfield=小说标题

# 如果找到匹配结果，自动设置：
# https://www.hetushu.com/book/38/24721.html
```

## 📊 爬取进度监控

### 实时反馈
- ✅ **成功消息**：显示爬取成功的章节数量
- ❌ **错误消息**：显示具体的错误信息
- ⏱️ **进度提示**：显示当前操作状态

### 日志格式
```
✅ 小说 "斗破苍穹" 爬取成功: 新增 2453 章
❌ 小说 "测试小说" 爬取失败: HTTP错误: 404
🔍 小说 "完美世界" 检测到来源链接: https://www.hetushu.com/book/1/123.html
```

```

```

#### 目录页链接
```

```

### 爬取规则
- **智能去重**：自动跳过已存在的章节
- **内容过滤**：只保存长度>50字的章节
- **错误容忍**：单个章节失败不影响整体爬取
- **进度保存**：支持断点续爬

## 🔧 故障排除

### 常见问题

#### 问题：未找到来源链接
```
解决步骤：
1. 确认小说标题正确
2. 检查网络连接
3. 尝试手动设置链接
4. 使用"自动检测"功能
```

#### 问题：爬取失败
```
解决步骤：
1. 检查来源链接是否有效
2. 确认网站是否支持爬取
3. 查看具体的错误信息
4. 尝试重新爬取
```

#### 问题：章节重复
```
解决步骤：
1. 系统会自动跳过重复章节
2. 检查章节标题是否一致
3. 查看爬取日志确认状态
```
