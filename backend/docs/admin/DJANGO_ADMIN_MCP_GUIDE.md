# Django Admin MCP功能使用指南

## 概述

现在您的Django Admin已经集成了MCP客户端功能，可以直接在管理界面中使用MCP Chrome工具进行章节数据管理。

## 功能说明

### 1. 现有功能
- **📖 AI修复章节信息**: 使用本地算法修复章节标题和序号

### 2. 新增MCP功能
- **🤖 MCP修复章节**: 使用MCP Chrome工具从网站获取最新章节信息并修复
- **🕷️ MCP爬取新章节**: 使用MCP Chrome工具从网站爬取新章节并添加到数据库

## 使用步骤

### 1. 访问Django Admin
```bash
cd backend
python manage.py runserver
```
访问：`http://localhost:8000/admin/`

### 2. 进入章节管理
1. 在Admin首页点击 **"Novels"** → **"Chapters"**
2. 或者直接访问：`http://localhost:8000/admin/novels/chapter/`

### 3. 使用MCP功能

#### 方法一：修复现有章节
1. 在章节列表中勾选需要修复的章节
2. 在"操作"下拉菜单中选择 **"🤖 MCP修复章节"**
3. 点击 **"执行"** 按钮
4. 系统将使用MCP Chrome工具获取网站数据并修复选中章节

#### 方法二：爬取新章节
1. 在章节列表页面（可选择或不选择章节）
2. 在"操作"下拉菜单中选择 **"🕷️ MCP爬取新章节"**
3. 点击 **"执行"** 按钮
4. 系统将从网站爬取新章节并添加到数据库

### 4. 查看结果
- 成功操作会显示绿色成功消息
- 失败操作会显示黄色警告消息
- 错误会显示红色错误消息

## 配置要求

### 1. MCP环境配置
确保您的系统已正确配置MCP Chrome工具：

```bash
# 检查MCP工具状态
mcp --version

# 确认Chrome浏览器可用
```

### 2. 替换模拟代码
在实际使用时，请将 `novels/views.py` 中的 `MCPClientService.get_page_content()` 方法中的模拟代码替换为真实的MCP调用：

```python
# 将模拟代码替换为：
result = mcp_chrome_get_web_content(url=url, htmlContent=True, textContent=False)
if result.get('success'):
    return result['data']['content'][0]['text']
else:
    print(f"获取失败: {result.get('error', '未知错误')}")
    return None
```

## 故障排除

### 常见问题

1. **"无法获取网站内容"**
   - 检查网络连接是否正常
   - 确认MCP Chrome工具已正确安装
   - 验证目标网站是否可访问

2. **"未找到章节数据"**
   - 检查网站结构是否变更
   - 确认解析规则是否正确
   - 联系技术支持

3. **操作执行缓慢**
   - 网络连接可能较慢
   - 目标网站响应时间较长
   - 考虑添加延时设置

### 调试方法

1. 查看Django Admin的错误消息
2. 检查服务器日志
3. 使用测试环境验证MCP配置
4. 联系技术支持团队
